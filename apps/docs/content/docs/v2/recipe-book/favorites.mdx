---
title: Favorites
description: Favorites endpoints for the Smart Recipe Book
---

<Callout variant="warning">
  These are authenticated endpoints. You can visit [authentication](../auth/register) to register an account.
</Callout>

These endpoints allow you to manage your favorited recipes. When you list favorites, the full recipe object is included so you don't need a second fetch. Favorites are unique per user and recipe — you cannot favorite the same recipe twice.

These endpoints support pagination and sorting. Read more about these features [here](../pagination-sorting).

## The Favorite model

<TypeTable
  type={{
    id: { type: "string", description: "The unique identifier for the favorite." },
    recipeId: { type: "string", description: "The ID of the favorited recipe." },
    recipe: {
      type: "Recipe",
      description: "The full recipe object."
    },
    owner: {
      type: "Profile",
      description: "The profile that favorited the recipe."
    },
    created: { type: "Date", description: "The date the recipe was favorited." }
  }}
/>

<EndpointDetails method="POST" path="/recipe-book/favorites" />

## Remove from favorites

<EndpointDetails method="DELETE" path="/recipe-book/favorites/<recipeId>" />

Remove a recipe from your favorites. Note that the URL parameter is the **recipe ID**, not the favorite ID.

Returns an empty `204 No Content` response on success.

<Callout variant="warning">
  If you have not favorited this recipe you will receive a `404 Not Found` error.
</Callout>
