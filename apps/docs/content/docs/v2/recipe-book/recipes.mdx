title: Recipes

<Callout variant="warning">
  Some of these are authenticated endpoints. You can visit [authentication](../auth/register) to register an account.
</Callout>

These endpoints allow you to create, read, update and delete recipes. Recipes are the core resource of the Smart Recipe Book — anyone can browse and read them, but creating, updating and deleting requires authentication and ownership.

These endpoints support pagination and sorting. Read more about these features [here](../pagination-sorting).

## The Recipe model

<TypeTable
  type={{
    id: { type: "string", description: "The unique identifier for the recipe." },
    title: { type: "string", description: "The title of the recipe." },
    description: { type: "string", description: "A description of the recipe." },
    prepTime: { type: "integer", description: "Preparation time in minutes." },
    cookTime: { type: "integer", description: "Cooking time in minutes." },
    servings: { type: "integer", description: "Number of servings." },
    difficulty: { type: "string", description: "The difficulty level (Easy, Medium, Hard)." },
    category: { type: "string", description: "The recipe category (e.g. Breakfast, Dinner, Pasta)." },
    ingredients: {
      type: "Array<object>",
      typeDescription: "Array of `{ name: string, quantity: number, unit: string }` objects.",
      description: "The list of ingredients."
    },
    instructions: {
      type: "Array<string>",
      description: "Ordered list of instruction steps."
    },
    tags: { type: "Array<string>", description: "A string array of tags." },
    image: {
      type: "object",
      typeDescription: "Instance of `Media` model.",
      description: "The image associated with the recipe."
    },
    owner: {
      type: "Profile",
      description: "The profile that created the recipe."
    },
    comments: {

      description: "Comments associated with this recipe."
    },

      type: "object",

    },
    created: { type: "Date", description: "The date the recipe was created." },

  }}
/>

## Update recipe

<EndpointDetails method="PUT" path="/recipe-book/recipes/<id>" />

Update a recipe by its id. Only the owner of the recipe can update it. You must provide at least one field to update.

> Please note that the `image.url` property must be a fully formed URL that links to a live and publicly accessible image. The API will check the provided URL and if it cannot be accessed publicly you will receive a `400 Bad Request` error response.

```json title="Request"
// You can update any combination of the properties of the recipe.
{
  "title": "Updated Margherita Pizza",
  "servings": 6,
  "tags": ["Italian", "Vegetarian", "Family"]
}
```

## Delete recipe

<EndpointDetails method="DELETE" path="/recipe-book/recipes/<id>" />

Delete a recipe by its id. Only the owner of the recipe can delete it. This will also delete all associated comments, favorites, and meal plan entries.

Returns an empty `204 No Content` response on success.
