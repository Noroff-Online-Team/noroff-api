This endpoint allows you to create, read, update and delete bookings.

This is a paginated and sortable endpoint. You can use the `limit` and `offset` query parameters to paginate the results. You can also use the `sort` and `sortOrder` query parameters to sort the results. Read more about the pagination and sorting features [here](../pagination-sorting).

## The Booking model

<TypeTable
  type={{
    id: { type: "string", description: "The unique id of the booking." },
    dateFrom: { type: "string", description: "The date the booking starts." },
    dateTo: { type: "string", description: "The date the booking ends." },
    guests: { type: "integer", description: "The number of guests for the booking." },
    created: { type: "string", description: "The date the booking was created." },
    updated: { type: "string", description: "The date the booking was last updated." }
  }}
/>

## Create entry

<EndpointDetails method="POST" path="/holidaze/bookings" />

Create a new booking.

```json title="Request"
{
  "dateFrom": "string", // Required - Instance of new Date()
  "dateTo": "string", // Required - Instance of new Date()
  "guests": 0, // Required
  "venueId": "string" // Required - The id of the venue to book
}
```

```json title="Response"
{
  "id": "string",
  "dateFrom": "string",
  "dateTo": "string",
  "guests": 0,
  "created": "string",
  "updated": "string"
}
```

<Hr />

## Update entry

<EndpointDetails method="PUT" path="/holidaze/bookings/<id>" />

Updating a booking.

```json title="Request"
{
  "dateFrom": "string", // Optional - Instance of new Date()
  "dateTo": "string", // Optional - Instance of new Date()
  "guests": 0 // Optional
}
```

```json title="Response"
{
  "id": "string",
  "dateFrom": "string",
  "dateTo": "string",
  "guests": 0,
  "created": "string",
  "updated": "string"
}
```

<Hr />

## Delete entry

<EndpointDetails method="DELETE" path="/holidaze/bookings/<id>" />

Delete a booking based on its id.

Returns an empty 204 response on success.
